Thorm Runtime Tests

Browser-native runtime checks without Node.js. Open a page to run its assertions in the browser.

Core Rendering

Basic client boot, events, conditional rendering, and keyed list reconciliation.

  • boot-basic.html Boots a minimal client tree and confirms initial DOM rendering.
  • events-inc.html Checks click listeners and atom increment updates.
  • show-toggle.html Verifies show mounts, unmounts, and remounts with atom changes.
  • repeat-reconcile.html Confirms keyed repeat rows reconcile content and order after updates.

Effects

Mount, watch, timeout, interval, and cleanup trigger behavior.

  • effect-on-mount.html Checks that mount effects run once after boot.
  • effect-watch.html Checks that watch effects react only after the watched value changes.
  • effect-after.html Checks timeout-triggered actions after the configured delay.
  • effect-every.html Checks repeated interval ticks while the tree remains mounted.
  • effect-timeout-dispose.html Checks disposed timeout effects do not fire after teardown.
  • effect-interval-dispose.html Checks disposed interval effects stop ticking after teardown.
  • effect-watch-dispose.html Checks disposed watch effects stop reacting to later changes.

HTTP

Successful requests plus failure-path handling for HTTP actions.

  • http-text.html Fetches text and stores both payload and status.
  • http-json.html Fetches JSON and renders the stored response body.
  • http-status.html Confirms deterministic response storage with status capture.
  • http-error-status.html Checks non-2xx responses still expose body and HTTP status.
  • http-error-json.html Checks invalid JSON falls back to null while preserving status.
  • http-error-network.html Checks failed fetches surface status -1 without mutating payload.

Routing

Matched routes, fallback behavior, params, query values, reroute flow, and route cleanup.

  • route-basic.html Confirms a matched route exposes param and query expressions.
  • route-fallback.html Checks fallback rendering when no route pattern matches.
  • route-query.html Checks query-string parsing through route query expressions.
  • route-params-multi.html Checks extraction across multiple dynamic path segments.
  • route-reroute.html Checks navigate triggers rerendering into the new route view.
  • route-dispose-restore.html Checks dispose restores the previous reroute hook and stops old rerenders.
  • route-view-cleanup-events.html Checks rerouted-away view handlers are disposed with the old route subtree.

Hydration

Hydration against SSR-like DOM for plain elements, conditionals, repeats, components, slots, and mismatch handling.

  • hydrate-basic.html Binds client events onto existing SSR-like markup.
  • hydrate-show.html Checks show hydration and later visibility toggles.
  • hydrate-show-mismatch.html Checks hydration fails clearly when visible show markup is missing.
  • hydrate-repeat.html Checks hydrated repeat rows stay interactive and reconcile correctly.
  • hydrate-repeat-mismatch.html Checks hydration fails clearly when keyed repeat rows are missing.
  • hydrate-component-slot.html Checks hydrated component props and projected slot content.
  • hydrate-slot-fallback-replace.html Checks hydrated projected slot content replaces fallback slot content.
  • hydrate-slot-fallback-empty.html Checks fallback slot content hydrates correctly when no projection exists.

Components

Prop evaluation, slot projection, nesting, and reactive prop flow through component trees.

  • component-props.html Checks component prop expressions and reactive prop updates.
  • component-slots.html Checks named slot projection and fallback rendering.
  • component-slot-replace.html Checks projected slot content replaces default fallback content.
  • component-nested.html Checks rendering across nested component boundaries.
  • component-props-nested.html Checks reactive prop updates reach deeply nested consumers.
  • component-named-slots-nested.html Checks named slots project through nested component composition.
